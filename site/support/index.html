常见问题

我的记录会自动上传吗？

不会。Morrow 采用本地优先设计，账户、交易、梦想、日记和图片保存在设备上。只有当你主动导出、分享，或明确授权使用第三方 AI 功能时，相关信息才可能离开设备。

怎样备份数据？

打开“设置”中的“备份与恢复”，生成 .gmmbackup 文件后，使用系统界面保存到 Files、iCloud Drive 或你信任的位置。备份可能含有私人内容，请妥善保管。

恢复备份会合并数据吗？

不会。通过完整校验并再次确认后，恢复操作会用备份中的账户、交易、梦想、日记、媒体和允许的普通偏好整体替换当前本地数据。恢复前建议先导出现有数据。

API Key 会进入备份吗？

不会。API Key 保存在 iOS Keychain 中；API Key、AI 服务配置和第三方授权均不进入 Morrow 备份。

怎样删除所有数据？

在设置页危险区选择“清除全部数据”，并完成两次确认。该操作会删除应用管理的记录、媒体、普通设置、AI 服务配置、API Key 和授权。你此前保存到 Files 或云盘的备份需要自行删除。

备份无法导入怎么办？

确认选择的是未被改名或修改的 Morrow .gmmbackup 文件，并确保设备有足够可用空间。为保护现有数据，校验和错误、缺失媒体、路径异常或不支持的版本都会被拒绝。仍无法解决时，请只发送错误提示，不要直接发送包含私人数据的备份。

Frequently asked questions

Are my records uploaded automatically?

No. Morrow is local-first. Accounts, transactions, dreams, diary entries, and images stay on your device unless you choose to export or share them, or explicitly grant access for a third-party AI feature.

How do I back up my data?

Open Backup & Restore in Settings, generate a .gmmbackup file, and use the system interface to save it to Files, iCloud Drive, or another location you trust. The backup may contain private content, so safeguard it carefully.

Does restore merge data?

No. After full validation and a second confirmation, restore replaces current local accounts, transactions, dreams, diary entries, media, and allowed ordinary preferences with the backup. Export current data first if you may need it later.

Is my API key included in backups?

No. API keys are stored in the iOS Keychain. API keys, AI provider configuration, and third-party grants are excluded from Morrow backups.

How do I delete everything?

Choose Clear All Data in the danger section of Settings and complete both confirmations. This removes app-managed records, media, ordinary settings, AI provider configuration, API keys, and grants. Backups previously saved to Files or cloud storage must be deleted separately.

Why was my backup rejected?

Confirm that it is an unchanged Morrow .gmmbackup file and that the device has enough free storage. To preserve current data, Morrow rejects checksum errors, missing media, unsafe paths, and unsupported versions. If the problem continues, email only the visible error message rather than the private backup.
